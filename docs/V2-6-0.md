Like2D was originally conceived as singleton-based.

This is no longer a good pattern.

See how the draw callback now has "g" as the first argument, with graphics?

Let's make it so that our whole system is managed this way. The first arg of every callback should be a "like" object with all the relevant API.

Also, the scene adapter shouldn't auto-call callbacks, actually. handleEvent should end, not by returning an event, but (by default) calling a handler that
automatically calls methods on the scene itself. That handler will be exposed so we can call it manually, in the case of custom handleEvent. 

This reduces the content of our adapters to wrappers around a single function handleEvent.

This should be exposed -- sophisticated devs can wire this up themselves, essentially writing their own adapter.

If that does mean managing state, that state should be centralized and easy to set up.

Finally, the "like" singleton in the callback architecture should be managed by the user.

They can export a variable to all of their own modules that contains a "like" instance which can be used accordingly.
