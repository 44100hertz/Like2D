# v2.7.0 Release Notes (Draft)

## Breaking Changes

### Scene Adapter: `runner.like` → `runner._like`

The internal `like` property on `SceneRunner` has been renamed to `_like` to indicate it's a private implementation detail.

**Migration:**
```typescript
// Before (v2.6.x)
class MyScene implements Scene {
  load(): void {
    this.runner.graphics.draw(...);  // Wrong API
    this.runner.like.gfx.draw(...);  // Accessing internal like
  }
}

// After (v2.7.0)
class MyScene implements Scene {
  load(like: Like): void {
    like.gfx.draw(...);  // Use the like parameter from callbacks
  }
}
```

Scene callbacks now receive `like` as the first parameter - use that instead of accessing `runner.like`.

## Rationale

Per the Chainsaw Principle (PHILOSOPHY.md:VIII): "We aren't kids. Just give me the chainsaw."

While we expose the wrench when you need it, the `like` instance is the core engine that scenes operate within. Making it "private" with the underscore convention signals that:
1. Direct access to the engine is advanced usage
2. Prefer using the provided callback parameters
3. We reserve the right to change internal structure

## Other Changes

### Transform API Added

The Canvas transform methods (`push`, `pop`, `translate`, `rotate`, `scale`) are being added to the curated graphics API. These wrap the native Canvas transform operations while maintaining our stateless principle - each draw call respects the current transform state.

**Rationale:**
Per the Chainsaw Principle (PHILOSOPHY.md:VIII): "We aren't kids. Just give me the chainsaw."

While we curate the API, transforms are fundamental to 2D graphics. Rather than forcing users to manage transformation matrices manually, we provide direct but controlled access to Canvas transforms.

**Usage:**
```typescript
// Apply transforms using push/pop to isolate state
like.gfx.push();
like.gfx.translate([100, 100]);
like.gfx.rotate(Math.PI / 4);
like.gfx.rectangle("fill", "red", [0, 0, 50, 50]);
like.gfx.pop();

// Subsequent draws are unaffected
like.gfx.rectangle("fill", "blue", [0, 0, 50, 50]); // No rotation applied
```
